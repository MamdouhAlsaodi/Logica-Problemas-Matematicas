# 🤖 claude.md — دليل استخدام Claude في مشروع منصة المسائل الرياضية

## ما هذا الملف؟

هذا الملف يوضح كيف يمكنك الاستفادة من Claude (الذكاء الاصطناعي) في كل مرحلة من مراحل تطوير مشروعك، من التخطيط وحتى التوثيق النهائي.

---

## 🧠 كيف يساعدك Claude في كل مرحلة؟

### المرحلة 1 — التخطيط وتصميم البيانات

**ما يمكن لـ Claude فعله:**
- اقتراح هيكل قاعدة البيانات وتصميم الجداول
- مقارنة التقنيات (Node.js vs Python، React vs Vanilla JS)
- رسم مخططات ERD نصية أو Mermaid لتوضيح العلاقات

**مثال على طلب:**
```
"ساعدني في تصميم جداول قاعدة بيانات لمنصة مسائل رياضية تحتوي على مستخدمين ومسائل بمستويات صعوبة مختلفة"
```

---

### المرحلة 2 — بناء الـ Backend

**ما يمكن لـ Claude فعله:**
- كتابة كود الـ API الكامل (Express / FastAPI)
- إنشاء نماذج قاعدة البيانات (Models)
- برمجة عمليات CRUD مع التحقق من البيانات (Validation)
- تحديد الأخطاء وإصلاحها

**مثال على طلب:**
```
"اكتب لي Route بلغة Node.js/Express يضيف مسألة رياضية جديدة مع التحقق من أن الحقول غير فارغة"
```

---

### المرحلة 3 — الاختبار بـ Postman

**ما يمكن لـ Claude فعله:**
- توليد بيانات تجريبية (Seed Data) واقعية
- شرح كيفية إعداد طلبات Postman خطوة بخطوة
- تحليل رسائل الخطأ وتقديم الحلول

**مثال على طلب:**
```
"أعطني 5 مسائل رياضية تجريبية بصيغة JSON لاختبار API الإضافة في Postman"
```

---

### المرحلة 4 — تطوير الواجهة (Frontend)

**ما يمكن لـ Claude فعله:**
- كتابة مكونات React أو HTML/JS كاملة
- ربط الواجهة بالـ API باستخدام `fetch` أو `axios`
- تصميم نماذج الإضافة والتعديل مع التحقق من المدخلات
- حل مشاكل CORS والمصادقة

**مثال على طلب:**
```
"اكتب مكوّن React يعرض قائمة المسائل الرياضية من API ويتيح حذفها وتعديلها"
```

---

### المرحلة 5 — اللمسات النهائية

**ما يمكن لـ Claude فعله:**
- مراجعة الكود وتحسين الأداء
- كتابة ملف README.md احترافي
- اقتراح تحسينات أمنية (مثل JWT، rate limiting)
- توليد تعليقات توثيقية (JSDoc / docstrings)

**مثال على طلب:**
```
"راجع هذا الكود وأخبرني إذا كان هناك ثغرات أمنية أو تحسينات مقترحة"
```

---

## 💡 نصائح للتعامل الفعّال مع Claude

| النصيحة | التفاصيل |
|---------|---------|
| **كن محدداً** | اذكر التقنية المستخدمة واسم الملف أو الدالة المحددة |
| **أرسل الكود الحالي** | ألصق الكود الذي تعمل عليه ليساعدك Claude بدقة أكبر |
| **اطلب خطوة بخطوة** | لا تطلب كل شيء دفعة واحدة، ابدأ بجزء صغير |
| **وضّح البيئة** | أخبر Claude إذا كنت تستخدم Windows أو Mac أو Linux |
| **اطلب التفسير** | إذا لم تفهم الكود، اطلب شرحاً مبسطاً |

## 📌 حدود Claude وما يجب أن تتولاه أنت

- **Claude لا يتذكر محادثات سابقة** — أعد تقديم السياق في كل جلسة
- **تحقق دائماً من الكود** — لا تنسخ وتلصق دون فهم ما يحدث
- **القرارات المعمارية الكبرى** — أنت من يقرر اتجاه المشروع، Claude يقترح فقط
- **بيانات حساسة** — لا ترسل كلمات مرور أو مفاتيح API حقيقية إلى Claude
